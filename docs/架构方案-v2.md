# 家庭药箱系统架构方案（v2 · MVP）

> v1 已归档在 `../家庭药箱系统整体技术架构方案.md`。
> v2 基于以下已确认前提收敛：
>
> - 复用**已有 NestJS（Prisma）后端** + **Vue3 + Naive UI 后台**
> - 认证走 JWT + Refresh Token，但 **v1 不做登录**（匿名/固定 devUserId）
> - Agent 单独起 **Python FastAPI + LangGraph** 服务
> - v1 **不做智能音响**、不做语音
> - v1 目标：**App 端输入症状 → 返回推荐药品信息**，一条最短闭环

---

## 一、v1 功能边界

### 做

- Tauri App 输入症状文本，调 NestJS `/consultation/ask`
- NestJS 转发给 Python Agent，Agent 调 LLM → 产出结构化药品推荐
- App 展示推荐药品卡片（名称、适应症、OTC/Rx、风险提示、免责声明）
- 后台管理新增 **药品库管理** + **会话日志查看** 两个菜单
- 所有 Agent 调用写入 `agent_traces`，方便调 prompt

### 不做（延后）

- 用户登录 / 注册 / 多用户隔离 → 全部挂到 `devUserId = "local-dev"`
- 用户家庭药箱 CRUD（App 里继续用 mock 数据展示 UI，但不落库）
- 语音输入 / 智能音响
- 图片识别 / 条码识别 的真实后端（UI 保留，提交后走 mock）
- 历史记录的多端同步（App 内存态即可）
- 风险规则引擎（v1 靠 LLM 自带的风险提示字段）

## 四、数据库设计（Prisma schema 草案）

放在已有 Prisma schema 里，用 `@@schema("family_doctor")` 做物理隔离（Prisma Preview: `multiSchema`）：

```prisma
// prisma/schema.prisma (增量)

generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["multiSchema"]
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
  schemas  = ["public", "family_doctor"]
}

model MedicineCatalog {
  id               String   @id @default(cuid())
  name             String
  aliases          String[] // 别名/通用名/商品名
  otc              OtcType  // OTC | RX
  indication       String   // 适应症
  contraindication String?  // 禁忌人群
  adverseReaction  String?  // 不良反应
  dosage           String?  // 用法用量
  barcode          String?  @unique
  approvalNumber   String?  // 国药准字
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt

  @@index([name])
  @@schema("family_doctor")
  @@map("medicine_catalog")
}

enum OtcType {
  OTC
  RX

  @@schema("family_doctor")
}

model ConsultationSession {
  id         String                @id @default(cuid())
  devUserId  String                // v1 固定 "local-dev"
  title      String?
  createdAt  DateTime              @default(now())
  messages   ConsultationMessage[]
  traces     AgentTrace[]

  @@index([devUserId, createdAt])
  @@schema("family_doctor")
  @@map("consultation_session")
}

model ConsultationMessage {
  id          String   @id @default(cuid())
  sessionId   String
  role        MsgRole  // USER | ASSISTANT
  content     String   @db.Text
  // ASSISTANT 回复附带的结构化推荐 (JSON 数组)
  recommends  Json?
  createdAt   DateTime @default(now())

  session     ConsultationSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)

  @@index([sessionId, createdAt])
  @@schema("family_doctor")
  @@map("consultation_message")
}

enum MsgRole {
  USER
  ASSISTANT

  @@schema("family_doctor")
}

model AgentTrace {
  id          String   @id @default(cuid())
  sessionId   String?
  nodeName    String   // parse / match / risk / render
  input       Json
  output      Json
  llmModel    String?
  tokenIn     Int?
  tokenOut    Int?
  latencyMs   Int
  error       String?
  createdAt   DateTime @default(now())

  session     ConsultationSession? @relation(fields: [sessionId], references: [id], onDelete: SetNull)

  @@index([sessionId, createdAt])
  @@index([createdAt])
  @@schema("family_doctor")
  @@map("agent_trace")
}
```

**命名约定**：实体用 `snake_case` 表名 + `camelCase` 字段，和已有后端风格保持一致；统一放 `family_doctor` schema 便于后续迁移/备份。

## 五、API 契约（v1 最小集）

### 1. `POST /consultation/ask`

App 主调接口，**同步响应**（v1 不做流式，等体验需要再升 SSE）。

**Request**

```json
{
  "sessionId": "optional-existing-session-id",
  "question": "这两天头痛发热，家里能吃什么药？"
}
```

**Response**

```json
{
  "sessionId": "ckxxx",
  "messageId": "ckxxx",
  "answer": "根据描述，可优先关注退热止痛类 OTC 药品，同时注意补水休息。若持续高热或加重请就诊。",
  "recommends": [
    {
      "medicineId": "ckmed_ibu",
      "name": "布洛芬缓释胶囊",
      "otc": "OTC",
      "indication": "头痛、发热、肌肉酸痛的短期缓解",
      "reason": "匹配\"头痛发热\"核心诉求",
      "warnings": ["孕晚期禁用", "胃溃疡慎用"]
    }
  ],
  "disclaimer": "本回复为 AI 参考建议，不构成医疗诊断。症状持续或加重请尽快就医。"
}
```

### 2. `GET /medicine/catalog?keyword=&page=&pageSize=`

App 药品查询页用，分页列表。

### 3. `GET /medicine/catalog/:id`

药品详情。

### 4. 后台管理端

- `POST /admin/medicine` / `PATCH` / `DELETE`
- `GET /admin/consultations?page=` + `GET /admin/consultations/:id/traces`

所有接口返回统一包装：

```ts
{ code: 0, data: T, message?: string }
```

与已有 NestJS `interceptor` 约定对齐（以你项目实际为准）。


`AgentClientService` 示例：

```ts
@Injectable()
export class AgentClientService {
  private http = axios.create({
    baseURL: this.config.get('AGENT_BASE_URL'),
    timeout: 30_000,
  });

  async consult(input: AgentConsultInput): Promise<AgentConsultOutput> {
    const { data } = await this.http.post('/agent/consult', input);
    return data;
  }
}
```

`ConsultationService.ask` 流程：

1. 若无 `sessionId` → 创建 session，`devUserId = 'local-dev'`
2. 插入 USER message
3. 查该用户可用药品（v1 = 全量 `MedicineCatalog`）
4. 调 `agentClient.consult({ question, medicines })`
5. 写 ASSISTANT message + 多条 `AgentTrace`
6. 返回给 App

## 七、Python Agent Service 骨架

新建仓库 `family-doctor-agent`：

```text
family-doctor-agent/
  pyproject.toml         # poetry/uv 二选一
  app/
    main.py              # FastAPI entry
    api/
      consult.py         # POST /agent/consult
    graph/
      builder.py         # LangGraph 构图
      nodes/
        parse.py         # 症状结构化
        match.py         # 药品召回+排序
        risk.py          # 风险提示
        render.py        # 组装最终响应
    llm/
      provider.py        # LLMProvider 抽象
      openai_provider.py
      deepseek_provider.py
    schemas.py           # Pydantic 模型, 对齐 NestJS DTO
    tracing.py           # 把每节点 IO 回传 NestJS / 或写 OTEL
    config.py
  tests/
    eval/
      cases.yaml         # 评测集: 症状 → 期望药品
      run_eval.py
```

**LangGraph 图（v1 直线版，不做条件分支）**

```
start → parse → match → risk → render → end
```

- `parse` 节点：LLM 结构化输出 `{ symptoms: [], severity, duration, populationHints }`
- `match` 节点：纯 Python，基于 `symptoms` 在传入的药品集合里做关键词/向量召回（v1 关键词即可）
- `risk` 节点：LLM 二次检查，针对每个候选药给出 `warnings`，带禁忌人群规则
- `render` 节点：LLM 生成最终 `answer` + `recommends` 结构

**接口**

```python
# POST /agent/consult
class ConsultRequest(BaseModel):
    sessionId: str
    question: str
    medicines: list[MedicineBrief]    # NestJS 传入可选药品清单

class ConsultResponse(BaseModel):
    answer: str
    recommends: list[Recommend]
    disclaimer: str
    traces: list[TraceStep]           # 每个节点的 IO, 供 NestJS 落库
```

**LLM Provider 抽象**

```python
class LLMProvider(Protocol):
    async def chat(self, messages: list[Message], **kw) -> str: ...
    async def structured(self, messages, schema: type[BaseModel], **kw) -> BaseModel: ...
```

v1 主用 DeepSeek（国内合规 + 便宜），OpenAI 作为备选；后续接通义/豆包只需新增 Provider。

## 十、合规与安全（v1 红线）

即便 MVP 也必须先做到：

1. **每条 AI 回复附强制免责声明**（`disclaimer` 字段，前端不能隐藏），用户可在设置里看到完整版。
2. **Rx 药处理**：Agent `render` 节点强制要求，若推荐包含 RX，必须追加"建议线下就诊并凭处方使用"提示；v1 可选择**直接过滤掉 RX，只返回 OTC**。
3. **敏感词兜底**：检测到自伤/自杀/急救类关键词，Agent 跳过常规链路，直接返回急救指引（120 / 当地急诊），**不做药品推荐**。
4. **数据出境**：症状文本如送 OpenAI 需脱敏；建议 v1 默认走 DeepSeek（国内）。
5. **日志留存**：`agent_trace` 保存 30 天以上，便于事后追溯与评测。

## 附录 A · 种子 Prompt 约定（供 Agent 使用）

**System Prompt 核心条款**（会随版本迭代，放 `family-doctor-agent/prompts/consult.v1.md` 管理）：

```
你是"家庭用药参考助手"，不是医生。你的输出必须：
1. 仅从用户传入的药品清单中挑选, 不得虚构。
2. 每个推荐必须给出: name, otc, reason, warnings[]。
3. 遇到 RX 药, 必须提示"建议线下就诊并凭处方使用"。
4. 遇到儿童/孕妇/哺乳/严重慢性病线索, 优先给风险提示, 推荐置后甚至不推荐。
5. 遇到自伤/呼吸困难/意识障碍等急症信号, 仅返回急救引导, 不给药品。
6. 每次回复必须附固定免责声明。
```
