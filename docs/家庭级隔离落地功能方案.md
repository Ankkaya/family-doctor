# 家庭级隔离落地功能方案

## 目标

将当前 `local-dev` 的单一示例药箱升级为按“家庭”隔离的数据模型。系统中的药品推荐、家庭药箱、问诊历史、设备绑定和后台查看都以 `householdId` 为主要归属边界，用户通过家庭成员关系访问同一个家庭药箱。

核心原则：

- 全局药品库只维护药品主数据，不表示某个家庭真实拥有该药。
- 家庭药箱库存归属家庭，同一家庭成员共享。
- 问诊推荐只从当前家庭药箱库存中选药。
- App 请求不信任前端传入的家庭归属，后端从登录态解析用户与当前家庭。
- 后台可按家庭、成员、药品、有效期和会话筛选追溯。

## 现状与问题

当前实现：

- 全局药品库：`medicine_catalog`
- 示例家庭药箱：`user_medicine_inventory`
- 问诊会话：`consultation_session.dev_user_id = local-dev`
- App 端未登录，默认读取 `local-dev` 药箱

当前问题：

- 不同 App 用户会共享同一份 `local-dev` 药箱。
- 后台只能看到 `devUserId`，不能识别家庭、成员或真实用户。
- 问诊历史无法按家庭隔离。
- 后续设备绑定、家庭成员风险画像无法自然扩展。

## 功能边界

本期做：

- App 用户身份。
- 家庭创建与当前家庭选择。
- 家庭成员关系。
- 家庭药箱库存。
- 问诊按家庭药箱推荐。
- 问诊会话绑定用户和家庭。
- 后台新增家庭、家庭成员、家庭药箱查看能力。
- 兼容迁移现有 `local-dev` 数据。

本期不做：

- 复杂家庭邀请审批流。
- 多家庭间药箱合并。
- 儿童、老人、孕妇等详细健康画像规则引擎。
- 药品真实监管数据同步。
- 智能音响设备接入。

## 角色与权限

App 端角色：

- 家庭拥有者：创建家庭、管理成员、管理药箱。
- 家庭成员：查看药箱、发起问诊、查看家庭问诊历史。

后台角色：

- 超级管理员：查看所有家庭、成员、药箱和问诊记录。
- 运营管理员：查看家庭药箱和问诊日志，不可删除家庭。
- 药品管理员：维护全局药品库，不直接修改家庭库存。

## 数据模型

### AppUser

App 用户身份，与后台 `User` 分离，避免把后台管理账号和患者端账号混用。

```prisma
model AppUser {
  id          String    @id @default(cuid())
  phone       String?   @unique
  email       String?   @unique
  nickname    String?
  avatarUrl   String?   @map("avatar_url")
  status      String    @default("active")
  createdAt   DateTime  @default(now()) @map("created_at")
  updatedAt   DateTime  @updatedAt @map("updated_at")
  deletedAt   DateTime? @map("deleted_at")

  memberships HouseholdMember[]
  consultations ConsultationSession[]

  @@index([phone])
  @@index([deletedAt])
  @@map("app_user")
}
```

### Household

家庭空间，是药箱与会话的隔离边界。

```prisma
model Household {
  id          String    @id @default(cuid())
  name        String
  ownerUserId String    @map("owner_user_id")
  createdAt   DateTime  @default(now()) @map("created_at")
  updatedAt   DateTime  @updatedAt @map("updated_at")
  deletedAt   DateTime? @map("deleted_at")

  members     HouseholdMember[]
  inventory   HouseholdMedicineInventory[]
  sessions    ConsultationSession[]

  @@index([ownerUserId])
  @@index([deletedAt])
  @@map("household")
}
```

### HouseholdMember

用户与家庭的成员关系。

```prisma
model HouseholdMember {
  id          String    @id @default(cuid())
  householdId String    @map("household_id")
  userId      String    @map("user_id")
  role        String    @default("member") // owner | member
  displayName String?   @map("display_name")
  joinedAt    DateTime  @default(now()) @map("joined_at")
  deletedAt   DateTime? @map("deleted_at")

  household   Household @relation(fields: [householdId], references: [id], onDelete: Cascade)
  user        AppUser   @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([householdId, userId])
  @@index([userId])
  @@index([deletedAt])
  @@map("household_member")
}
```

### HouseholdMedicineInventory

家庭真实拥有的药品库存。它引用全局药品库。

```prisma
model HouseholdMedicineInventory {
  id          String          @id @default(cuid())
  householdId String          @map("household_id")
  medicineId  String          @map("medicine_id")
  quantity    Int             @default(1)
  expireAt    DateTime?       @map("expire_at") @db.Date
  source      String?         // manual | image | barcode | admin
  notes       String?
  createdBy   String?         @map("created_by")
  createdAt   DateTime        @default(now()) @map("created_at")
  updatedAt   DateTime        @updatedAt @map("updated_at")
  deletedAt   DateTime?       @map("deleted_at")

  household   Household       @relation(fields: [householdId], references: [id], onDelete: Cascade)
  medicine    MedicineCatalog @relation(fields: [medicineId], references: [id], onDelete: Restrict)

  @@unique([householdId, medicineId])
  @@index([householdId])
  @@index([medicineId])
  @@index([expireAt])
  @@index([deletedAt])
  @@map("household_medicine_inventory")
}
```

### ConsultationSession 调整

```prisma
model ConsultationSession {
  id          String   @id @default(cuid())
  userId      String?  @map("user_id")
  householdId String?  @map("household_id")
  devUserId   String?  @map("dev_user_id")
  title       String?
  createdAt   DateTime @default(now()) @map("created_at")

  user        AppUser?   @relation(fields: [userId], references: [id], onDelete: SetNull)
  household   Household? @relation(fields: [householdId], references: [id], onDelete: SetNull)
  messages    ConsultationMessage[]
  traces      AgentTrace[]

  @@index([householdId, createdAt])
  @@index([userId, createdAt])
  @@map("consultation_session")
}
```

过渡期保留 `devUserId`，迁移完成后可删除。

## 认证与当前家庭

App 请求必须携带 token。

```http
Authorization: Bearer <app_access_token>
X-Household-Id: <household_id>
```

后端处理规则：

1. 从 JWT 解析 `appUserId`。
2. 若请求带 `X-Household-Id`，校验该用户是该家庭成员。
3. 若未带，读取用户默认家庭。
4. 所有家庭药箱和问诊接口都使用校验后的 `householdId`。

MVP 可以先采用匿名用户：

- App 首次启动调用 `POST /app/auth/anonymous`。
- 后端创建 `AppUser` 和默认 `Household`。
- App 保存 token。
- 后续可绑定手机号升级账号。

```http

```

```json
  "refreshToken": "...",

    "nickname": "家庭用户"

  "defaultHousehold": {
    "id": "house_xxx",
    "name": "我的家庭"
  }
}
```


### 新增家庭库存

```http
POST /medicine/cabinet
{
  "medicineId": "med_ibu",
  "quantity": 1,
  "expireAt": "2026-12-31",
  "source": "manual",
  "notes": "家庭常备"
}
```

### 更新家庭库存

```http
PATCH /medicine/cabinet/:inventoryId
{
  "quantity": 2,
  "expireAt": "2026-12-31",
  "notes": "剩余两盒"
}
```

### 删除家庭库存

```http
DELETE /medicine/cabinet/:inventoryId
```

### 发起问诊

```http
POST /consultation/ask
Authorization: Bearer ...
X-Household-Id: house_xxx
{
  "sessionId": "optional",
  "question": "头痛发热两天"
}
```

后端流程：

1. 解析 `appUserId`。
2. 解析并校验 `householdId`。
3. 写入 `consultation_session.user_id` 和 `household_id`。
4. 从 `household_medicine_inventory` 查询候选药。
5. 只把该家庭药箱药品传给 Agent。
6. 保存 assistant 消息和 trace。

### 问诊历史

```http
GET /consultation/sessions?page=1&pageSize=20
GET /consultation/sessions/:id
```

只返回当前家庭可访问的会话。

## 后端模块调整

新增模块：

```text
backend/src/domains/app-auth/
backend/src/domains/households/
backend/src/domains/family-doctor/cabinet/
```

建议职责：

- `AppAuthModule`：App 匿名登录、手机号登录、refresh token。
- `HouseholdsModule`：家庭、成员关系、当前家庭校验。
- `CabinetModule`：家庭药箱库存 CRUD。
- `ConsultationModule`：问诊会话和 Agent 调用。
- `MedicineModule`：全局药品库。

共享工具：

```ts
resolveCurrentHousehold(req): Promise<{
  appUserId: string;
  householdId: string;
}>
```

所有 App 侧家庭资源接口都使用该方法。

## 数据迁移

### 第一步：新增表

- `app_user`
- `household`
- `household_member`
- `household_medicine_inventory`
- `consultation_session.user_id`
- `consultation_session.household_id`

### 第二步：迁移 local-dev

创建一个默认用户和默认家庭：

```text
AppUser: local-dev-user
Household: local-dev-household
HouseholdMember: owner
```

迁移现有库存：

```sql
INSERT INTO household_medicine_inventory (...)
SELECT
  'local-dev-household',
  medicine_id,
  quantity,
  expire_at,
  source,
  notes
FROM user_medicine_inventory
WHERE dev_user_id = 'local-dev';
```

迁移会话：

```sql
UPDATE consultation_session
SET user_id = 'local-dev-user',
    household_id = 'local-dev-household'
WHERE dev_user_id = 'local-dev';
```

### 第三步：切换代码读写

- `/medicine/cabinet` 改读 `household_medicine_inventory`。
- `/consultation/ask` 改用 `householdId` 查询候选药。
- `/consultation/sessions` 按 `householdId` 过滤。

### 第四步：保留回滚窗口

保留旧表 `user_medicine_inventory` 一个版本周期，只读不写。确认稳定后删除。

## 安全要求

- App 端不能直接传 `userId` 作为归属依据。
- `householdId` 必须经过成员关系校验。
- 后台家庭药箱修改必须记录操作日志。
- 删除家庭或库存使用软删除。
- 问诊 trace 不暴露给普通 App 用户，只给后台查看。
- 家庭药箱和问诊历史接口禁止跨家庭访问。

## 验收标准

### App

- 新用户首次启动自动创建默认家庭。
- 同一用户切换家庭后，药箱和历史随家庭变化。
- A 家庭录入的药，B 家庭不可见。
- 问诊推荐只来自当前家庭药箱。
- 问诊历史只展示当前家庭会话。

### 后端

- 所有 App 家庭资源接口从 token 解析用户。
- 所有家庭资源接口校验成员关系。
- `POST /consultation/ask` 的 Agent trace 中 `medicineCount` 等于当前家庭药箱候选数。
- 非家庭成员访问 `householdId` 返回 403。

### 后台

- 可按家庭筛选药箱库存。
- 可按家庭筛选问诊日志。
- 会话详情展示家庭、发起用户、推荐药品和 trace。

## 分阶段落地

### P0：数据结构与迁移

- 新增家庭级表。
- 迁移 `local-dev` 数据。
- 保留旧接口兼容。

### P1：App 匿名身份

- App 首次启动匿名登录。
- 后端签发 App token。
- App 请求带 token。

### P2：家庭药箱读写

- `/medicine/cabinet` 切换到 `household_medicine_inventory`。
- 手动录入写入后端。
- 药品列表完全来自家庭库存。

### P3：问诊隔离

- `/consultation/ask` 按当前家庭取候选药。
- 会话绑定 `userId + householdId`。
- 历史按家庭过滤。

### P4：后台管理

- 家庭列表。
- 家庭药箱列表。
- 问诊日志增加家庭和用户筛选。

### P5：手机号登录与家庭成员

- 匿名用户绑定手机号。
- 家庭邀请和成员管理。
- 多家庭切换。
