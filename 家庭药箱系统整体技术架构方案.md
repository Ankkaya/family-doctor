# 家庭药箱系统整体技术架构方案（v1）

---

## 一、项目目标

构建一个支持 **手机 App + 智能音响** 的家庭药箱智能问药系统：

- 用户通过文本或语音描述症状
- 系统基于用户家庭药箱进行药品匹配
- 返回推荐药品 + 用药说明 + 风险提示

---

## 二、总体架构

```text
客户端层
├── Tauri App（React）
├── 后台管理（Vue3 + Naive UI）
└── 智能音响设备

接入层
└── Next.js API（BFF）

服务层
├── Agent Service（LangGraph）
├── Voice Gateway（语音服务）
└── LLM Adapter

数据层
├── PostgreSQL
└── Redis
```

## 四、系统分层说明

### 1. 客户端层

#### Tauri App

功能：
- 症状输入（文本/语音）
- 问诊结果展示
- 家庭药箱管理
- 历史记录
- 设备绑定

#### 后台管理

功能：
- 药品库管理
- 风险规则管理
- 用户管理
- 会话日志

---

### 2. 接入层（Next.js）

职责：
- 用户认证
- API 网关
- 调用 Agent 服务
- 聚合返回结果

---

### 3. Agent 服务（核心）

基于 LangGraph 构建：

流程：

```text
输入 → 症状解析 → 用户画像 → 药箱过滤 → 风险校验 → 推荐生成 → 输出
```

模块：
- symptom-parser
- medicine-matcher
- risk-checker
- response-generator

---

### 4. 语音服务（Voice Gateway）

职责：
- 语音转文本
- 文本转语音
- 设备接入

---

### 5. 模型层

设计统一接口：

```ts
interface LLMProvider {
  chat()
  structured()
}
```

实现：
- OpenAIProvider
- DeepSeekProvider

---

## 五、核心业务模块

### 1. 问诊模块（consultation）
